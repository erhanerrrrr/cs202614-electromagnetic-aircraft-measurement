## 2. CST 仿真总体策略

CST 在本项目中承担“可信电磁数据源”的角色，不替代自研重建算法。

### 2.1 CST 负责

1. 建立标准源、多源组合、简化航空载体几何。
2. 设置频点、材料、边界条件、激励端口和监视器。
3. 当前在半球面 2π 测点上导出近场复数数据。
4. 导出 CST 自带远场方向图作为参考真值。
5. 生成仿真模型图、测点布局图、方向图可视化素材。

### 2.2 Python 负责

1. 读取 CST 导出的近场复数数据和远场真值。
2. 构建近场传播矩阵或等效源反演模型。
3. 进行 Tikhonov/SVD/L1/组稀疏反演。
4. 由等效源外推远场并与 CST 真值比较。
5. 做测点删减、测点优化和误差排序。
6. 提取空-频-极化特征并训练识别模型。

## 3. CST 模型分级

### Level 1：标准源验证

目的：验证 CST 导出数据和 Python 重建流程正确。

建议模型：

1. 短偶极子。
2. 半波振子。
3. 小环天线或简单贴片天线。

输出：

- 近场采样数据：Ex/Ey/Ez 或探头极化响应，复数幅相。
- 远场真值：Etheta/Ephi 或增益方向图。
- 解析方向图对照：短偶极子可用 sin(theta) 型方向图作为 sanity check。

通过标准：

- Python 重建远场与 CST 远场相关系数高。
- 主瓣方向误差小。
- 数据坐标、单位、相位约定明确。

### Level 2：多源、多频、多极化

目的：模拟航空载体上多设备协同辐射。

建议模型：

1. 两到四个辐射源，位置分布在机身前后、左右、顶部/腹部。
2. 多频点：建议先选 3 到 5 个离散频点，后续再扩展扫频。
3. 多状态：通信源开、雷达源开、导航源开、多源同时工作。
4. 多极化：至少水平/垂直或 theta/phi 两正交极化。

输出：

- 每个 sample_id 对应一个“源组合 + 频点 + 状态 + 测点集”。
- 形成识别数据集初版。

### Level 3：简化航空载体安装效应

目的：体现复杂载体结构、电气布局和安装位置影响。

建议几何：

1. 金属圆柱或椭圆柱机身，长度约 12 m。
2. 平板或简化翼型机翼，横向宽度约 10 m。
3. 垂尾/平尾简化结构。
4. 天线/辐射源安装在机身顶部、腹部、翼尖或侧向。

材料：

- 初期可用 PEC 或铝等高导电金属近似。
- 后续可加入介质罩、复合材料区域，作为加分项。

输出：

- 安装效应导致的方向图遮挡、散射、副瓣变化。
- 同源不同安装位置的空间辐射指纹差异。

### Level 4：扰动与鲁棒性

目的：证明方案不是只对理想数据有效。

扰动项：

1. 测量噪声：20/30/40 dB SNR。
2. 测点缺失：随机缺测、局部遮挡缺测。
3. 频点偏移：中心频率轻微偏移。
4. 姿态变化：俯仰/滚转/偏航小角度变化。

输出：

- 重建鲁棒性曲线。
- 识别准确率随噪声、缺测、姿态扰动变化曲线。

## 4. 测量布局设定

赛题要求：覆盖 2π 空间立体角的半柱面或半球面空间区域，可容纳尺寸不小于 12 m x 10 m x 8 m。

当前执行选择半球面；半柱面作为后续扩展备选，不进入本轮 CST Level 1/Level 2。

### 4.1 半球面方案

坐标系：

- 原点放在被测载体地面投影中心或几何中心，需要在报告中固定。
- 被测空间包络：x in [-6, 6] m，y in [-5, 5] m，z in [0, 8] m。
- 半球半径建议 R >= 13 m，覆盖最远角点 sqrt(6^2 + 5^2 + 8^2) = 11.18 m，并留有安装余量。

采样：

- theta：从顶部小角度到接近水平面，例如 5 deg 到 85 deg。
- phi：0 deg 到 360 deg。
- 初始点数：例如 9 x 18 = 162 个空间点。
- 每点双极化：等效 324 条测量响应。

### 4.2 半柱面方案

坐标系：

- 柱轴沿机身 x 方向。
- x 覆盖 [-7, 7] m，留出机身前后余量。
- 半径建议 R >= sqrt(5^2 + 8^2) + margin，取 10 m 左右更稳妥。

采样：

- x 方向均匀采样。
- 弧向覆盖 180 deg。
- 每点双极化或三分量场。

适用性：

- 半柱面更贴近大尺寸航空载体工程布设。
- 半球面更适合文献中的球面波展开和压缩感知迁移。

## 5. CST 导出数据规范

所有 CST 数据导出后统一转为 CSV/NPZ/HDF5，字段建议如下：

| 字段 | 含义 |
|---|---|
| sample_id | 样本编号，同一状态/频点/布局的唯一编号 |
| sensor_id | 测点编号 |
| x_m, y_m, z_m | 测点坐标，单位 m |
| theta_deg, phi_deg | 球坐标角度 |
| frequency_hz | 频率 |
| polarization | Ex/Ey/Ez 或 theta/phi/H/V |
| e_real, e_imag | 电场复数值 |
| h_real, h_imag | 磁场复数值，可选但建议保留 |
| source_config | 源组合编号 |
| carrier_model | 标准源/多源/简化载体模型编号 |
| working_state | 工作状态标签 |
| cst_project | CST 工程文件名 |
| monitor_name | CST 场监视器名称 |

注意：

1. 必须保存相位信息；如果只能导出幅值，则需要走无相位重建备用路线，难度更高。
2. 坐标系和单位要固定，不能每个模型单独解释。
3. 近场数据和远场真值必须用同一个 sample_id 对齐。
4. 每次导出保留 CST 工程截图和参数表，便于报告追溯。

## 6. 什么时候需要三人分工

当前阶段先独立推进文献、方案和 baseline。建议到以下节点再分工：

1. CST 标准源模型需要真实操作时。
2. Python baseline 读入字段稳定后，需要批量 CST 数据时。
3. 多源/载体数据量变大，需要一个人专门维护数据字典和识别实验时。
4. 报告/PPT/视频进入成稿阶段时。
