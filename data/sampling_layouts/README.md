# Sampling Layouts

This directory stores small, versionable candidate sensor layouts for the 2pi
hemisphere measurement workflow. The CSV files are generated by:

```powershell
python code\optimize_sampling_layout.py
```

## Files

| File | Purpose |
|---|---|
| `hemisphere_sampling_candidates.csv` | One row per selected sensor in each candidate layout. |
| `sampling_layout_summary.csv` | Reconstruction, matrix, spatial, and classification proxy metrics. |
| `sampling_layout_summary.json` | Machine-readable summary metadata for scripts and dashboards. |
| `cst_level1_tradeoff/` | Real CST Level 1 diagnostic using a 5 x 3 x 3 equivalent-source grid. |
| `cst_level1_center_source_check/` | Level 1 center-source calibration check at `(0, 0, 4 m)`. |
| `cst_level1_source_model_sweep/` | Level 1 equivalent-source grid and regularization sweep. |
| `cst_level1_sparse_calibration/` | Level 1 group-sparse equivalent-source calibration. |
| `cst_level1_convention_check/` | Level 1 phase, conjugation, and polarization convention diagnostic. |
| `spherical_nf_ff_baseline/` | Lightweight spherical-harmonic NF-FF sanity baseline independent of equivalent-source grids. |

These results are synthetic equivalent-source proxy evidence. They select the
next CST export cases; they do not replace validation on real CST near-field
and far-field CSV data.

The CST Level 1 result folders are generated by `code/run_cst_sampling_tradeoff.py`,
`code/run_cst_source_model_sweep.py`, `code/run_cst_sparse_reconstruction.py`,
`code/run_cst_level1_convention_check.py`, and
`code/run_spherical_nf_ff_baseline.py`. Read
`docs/cst_level1_sampling_calibration_findings.md` before using those tables in
reports: the center-source check and spherical baseline validate the data path,
while the generic equivalent-source grid still needs source-prior and model
calibration before final sampling claims.
